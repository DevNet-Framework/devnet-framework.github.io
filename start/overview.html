Overview

Devnet framework is free open-source, PHP software and cross-platform, with high-performance, for building console and modern web application, with rapid application development tools.

Server requirements

Befor installing DevNet Framework, make sure your server meets the following requirements :

  • PHP 7.4 or higher version
  • PDO PHP Extension
  • Composer Dependency Manager

You can get the latest version of PHP from php.net
also you can get Composer the PHP Dependency Manager from getcomposer.org
and you may need Git the distributed version control system from git-scm.com


Installing DevNet Cli

devnet/cli is a command-line Interface tool, that used for creating and running DevNet projects.

In order to install DevNet Cli Globally, and Because DevNet is still in development, we need first to config composer global minimum-stability to development.

So in the terminal type the following command: 


composer global config minimum-stability dev

Then force composer to prefer the stable version if is was available, like so: 


composer global config prefer-stable true

To Install DevNet Cli globally, run the following command in the terminal: 


composer global require devnet/cli

This will run a minimal installation of DevNet Framework that allows you to create a console app only, but optionally you can add other DevNet packages later, like Web Components and Entity ORM.

Optionally you can extend DevNet Cli by installing DevNet templates add-ons, to be able to create projects other than console projects like Web API and MVC projects, using the DevNet command-line.

So in your terminal you can add DevNet Templates like this: 


composer global require devnet/templates

Using DevNet Command:

For the first start you can run the devnet command with the option --version in your terminal, to check the DevNet Cli version and see if it was installed properly, like so: 


devnet --version
Output

DevNet command-line interface v1.0.0
Copyright (c) DevNet

Note: Before using DevNet command make sure that the path of vendor/bin of your composer already added to your Environment Variables System, for example in Linux you must add this line to your system:
export PATH="$PATH:$HOME/.config/composer/vendor/bin"

You can run also the devnet command with the option --help in your terminal, to show the help on how to use devent command-line tool, as shown below: 


devnet --help
Output

DevNet command-line interface v1.0.0
Usage: devnet [options]

Options:
   --help      Show command line help.
   --version   Show DevNet Cli version.
   --path      Show DevNet runtime path.

Usage: devnet [command] [arguments] [options]

commands:
   new   Create a new project
   run   Run the DevNet applicaton
   add   Add a template code file to the project

Run 'devnet [command] --help' for more information on a command.

The Help shows a list of available commands and options that you can use with DevNet Cli.

For more details about a command that you want to use, you can run it with the option --help like in the following example: 


devnet new --help
Output

Usage: devnet new [template] [arguments] [options]

Options:
   --help     Displays help for this command.
   --project  Location to place the generated project.

templates:
   console    Console Applicatinon project
   mvc        DevNet MVC Web Application Project
   web        DevNet Web Application Project

The help shows a usage of the new command followed by the name of the project template that you want to create, and followed by options and arguments, and shows a list of options that you can use with the command and a list of templates that you can choose from to create your project.


Uninstalling DevNet

To uninstall DevNet Cli globally from your system, run the following command in the terminal: 


composer global remove devnet/cli
