Get Started

Entity ORM (Object-relational Mapping) is a system that offers an automated mechanism to developers for accessing and storing the data in the database in an object-oriented way, by performing the work required to map between Entities defined in an application's programming language as classes and data stored in relational data sources as tables, without needing the data-access code to be written.

The following diagram is an abstraction of Entity ORM architecture.



EntityContext: is an implementation of the Unit of work pattern, represents a gateway to the database by providing an API to communicate with the database with the following capabilities:

  • Database Connections
  • Data querying and persistance
  • Change Tracking
  • Model building
  • Data Mapping
  • Transaction management

EntitySet: is an implementation of the Repository pattern, a class that holds a collection of entities of the same type related to a table in the database, and each entity is an instance that represents a data row in that table.


Installation

If the DevNet framework is not fully installed, you can install the Entity ORM package into your project using Composer by running the following command in your terminal: 

composer require devnet/entity

Defining the EntityContext

The recommended way to work with Entity ORM is to define a context class in the "Models" folder that derives from DevNet\Entity\EntityContext and exposes DevNet\Entity\EntitySet properties that represent collections of the specified entities in the context, as shown in the following example. 

<?php

namespace Application\Models;

use DevNet\Entity\EntityContext;
use DevNet\Entity\EntityOptions;
use DevNet\Entity\EntitySet;

class BlogContext extends EntityContext
{
   public readonly EntitySet $Posts;

   public function __construct(EntityOptions $options)
   {
      parent::__construct($options);
      $this->Posts = $this->set(Post::class);
   }
}

Entity Type

By convention, the entity types that are exposed in EntitySet properties in EntityContext are included in the model and mapped automatically to the database tables based on their shape.

  • Each entity class will be mapped automatically to the corresponding database table with the same name.
  • Each public property will be mapped to the corresponding column with the same name in the table.
  • By default, the property that is named "Id" will be considered as the primary key.

The following code example is the entity class Application\Models\Post used in the previous context, and it represents the Post table in the database where the properties represent the columns of that table. 

<?php

namespace Application\Models;

use DateTime;

class Post
{
   public int $Id;
   public int $UserId;
   public string $Title;
   public string $Content;
   public DateTime $Date;
}

Data annotations

Data annotations are attributes that can be applied to classes and properties to provide additional information about how they should be mapped to the database. By using data annotations, we can specify the table name and column names that should be used when mapping the entity to the database.

Assuming the entity Post presented previously has a property used as the primary key that does not follow the naming convention mentioned earlier. Instead of Post::Id, it has Post::PostId. The corresponding table in the database has a different naming. In this case, we can use data annotations to map the entity without changing its shape, like this in the following example: 

<?php

namespace Application\Models;

use DevNet\Entity\Annotations\Column;
use DevNet\Entity\Annotations\Primarykey;
use DevNet\Entity\Annotations\Table;
use DateTime;

#[Table('Posts')]
#[Primarykey('PostId')]
class Post
{
   #[Column('post_id')]
   public int $PostId;

   #[Column('user_id')]
   public int $UserId;

   #[Column('title')]
   public string $Title;

   #[Column('content')]
   public string $Content;

   #[Column('date')]
   public DateTime $Date;
}

Connecting to the Database

The simplest way to connect Entity ORM to the database, is by creating an instance of the derived class of EntityContext, and passing the database configuration options to the constructor as follows: 

<?php

use Application\Models\BlogContext;
use DevNet\Entity\MySql\MySqlDataProvider;
use DevNet\Entity\EntityOptions;

$options = new EntityOptions();
$options->ProviderType = MySqlDataProvider::class;
$options->ConnectionString = "//user:password@127.0.0.1/blog";

$blog = new BlogContext($options);
$posts = $blog->Posts;

Data Providers

The following table shows a list of database providers that Entity ORM supports with the corresponding connection string to each one.

Driver Data Provider Connection String
MySql DevNet\Entity\MySql\MySqlDataProvider "//username:password@127.0.0.1:3306/database"
PostgreSql DevNet\Entity\PgSql\PgSqlDataProvider "//username:password@127.0.0.1:5432/database"
SQLite DevNet\Entity\Sqlite\SqliteDataProvider "//path/to/database"

Connection String

The connection string used by the database providers follows the following url format:
"//[username]:[password]@[host]:[port]/[database]?[option=value]"


Registering the EntityContext

You can register Entity ORM as a reused service that will be injected into your application using the Extension method ServiceCollectionExtensions::addEntityContext() inside the method WebHostBuilder::register(), as shown in the following code example: 

<?php

namespace Application;

use DevNet\Web\Hosting\WebHost;
use DevNet\Web\Extensions\ApplicationBuilderExtensions;
use DevNet\Web\Extensions\ServiceCollectionExtensions;
use DevNet\Entity\MySql\MySqlDataProvider;
use Application\Models\BlogContext;

class Program
{
   public static function main(array $args = [])
   {
      $builder = WebHost::createDefaultBuilder($args);
      $builder->register(function ($services) {
         // adding EntityContext
         $services->addEntityContext(BlogContext::class,
            function ($options) {
               $options->ProviderType = MySqlDataProvider::class;
               $options->ConnectionString = "//user:password@127.0.0.1/blog";
            }
         );
      });
      ...
   }
}

You can then get the EntityContext using the constructor injection to add the EntityContext to the controller class and use it in the controller methods like the following example: 

<?php
                  
namespace Application\Controllers;

use DevNet\Web\Endpoint\ActionController;
use DevNet\Web\Endpoint\IActionResult;
use DevNet\Web\Endpoint\Route;
use Application\Models\BlogContext;

class BlogController extends ActionController
{
   private BlogContext $blog;

   public function __construct(BlogContext $blog)
   {
      $this->blog = $blog;
   }

   #[Route("/")]
   public function index(): IActionResult
   {
      $this->view(['posts' => $this->blog->Posts]);
   }
}
